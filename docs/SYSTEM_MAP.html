שלושת התפקידים

טכנאי

permissions = 'technician'
  • צופה בקריאות שלו לפי profession_name
  • פותח קריאה חדשה לכלי
  • מבקש חלקים (מוסיף שורות ל-call_required_parts)
  • סוגר קריאה
  • ממלא טופס "דרישת כיתה"
  • מעדכן מיקום/מחלקה לכלי
  • מדווח שעות מנוע + ק"מ לטנק

מחסנאי

permissions = 'warehouse'
  • מקדם פריטים בפייפליין (כל המעברים בין סטטוסים)
  • מנפק לטכנאי דרך part_withdrawals
  • מנהל קטלוג חלקים
  • יוצר "הזמנת מחסן כללית"
  • מעדכן מספר דרישה / מק״ט
  • מטפל ב"לא נצרך" → "במלאי"

מנהל

permissions = 'manager'
  • שליטה מלאה — שינוי profession_name, מחיקה, ביטול
  • תור חריגות (קריאות בלי כלי ידוע)
  • מנהל עובדים / כלים / מקצועות / זמינות
  • טבלת "דרישות כיתות" עם כפתור העתקה ל-WhatsApp
  • טבלת "שעות מנוע / קילומטר" + סימון שבוע חודשי לטנקים
  • דוח נוכחות יומי בלחיצה

כפתורי העתקה מהירה ⧉

המערכת מבוססת בכבדות על העתקה ל-WhatsApp/SMS. הכפתורים פזורים על פני הדפים.

תפריט העתקה בכל שורת פריט

CopyMenu · בטבלאות פעולות פתוחות, נדחו, חסומים, נופקו, לא נצרך

סמליל ⧉ פותח תפריט עם שלוש אופציות: פורמט מלא ל-WhatsApp, רק ה-מק״ט, רק מספר הדרישה. אישור צף במרכז המסך ל-0.8 שניות.

דוח נוכחות יומי

AttendanceReportButton · ManagerHomePage

בלחיצה — מועתק טקסט "*חימוש DD.MM*" + רשימת עובדים מקובצת לפי מקצוע עם "נוכח"/"בבית" לכל אחד.

דרישת כיתה

ClassOrdersTable · ManagerHomePage

על כל שורת דרישת כיתה — כפתור "העתק טקסט" יוצר את הפורמט הקבוע של "דרישת כיתת אחזקה" מוכן ל-WhatsApp.

פאנל "הזמן כיתה"

OrderClassPanel · CallDetailPage (מנהל) ו-NewCallForm

טכנאי שומר. מנהל רואה כפתור "העתק טקסט" שמפיק את אותו פורמט מתוך כרטיסיית הקריאה.

תקציר קריאה

CopyCallSummaryButton · CallDetailPage

"היי, זה X, לגבי קריאה Y עבור כלי Z..." — להדבקה בשיחה עם המדווח.

פורמט פרטים מלא לחלק

buildCopyText · משמש את CopyMenu

בונה את הטקסט הסטנדרטי של היחידה: *חטיבה:* X · *גדוד:* Y · *סוג הכלי:* ... · *מק״ט:* ... · *שם החלק:* ... · *כמות:* ...

צ׳/פלאפון/לינק לקריאה

פזור בכרטיסי קריאה ושורות פעולות פתוחות

כפתורי טלפון (חיוג + WhatsApp) + לינקים לקריאות, חלקם עם אינדיקציית "✓ הועתק" צפה.

קשרים "רכים" (טקסט, לא Foreign-Key) — שלושה במכוון:
  • vehicles.type_name ↔ professions.name — סוג כלי = שם מקצוע, אבל הקישור הוא טקסט פתוח כדי לאפשר עריכה גמישה.
  • employees.profession_name ↔ professions.name — אותו הדבר עבור התמחות העובד.
  • service_calls.vehicle_number ↔ vehicles.vehicle_number — במכוון פתוח. קריאות מ-Base44 עם מספר כלי לא ידוע ייקלטו ותהיה לגביהן חריגה דחופה למנהל, במקום להידחות. זה העיקרון הבסיסי של המערכת: "אין שדות חוסמים".
